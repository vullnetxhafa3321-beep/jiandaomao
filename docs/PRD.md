## 1. 背景与机会

### 1.1 问题陈述

城市中有大量流浪猫，路人「捡到猫」是高频但高度碎片化的场景。现有渠道（朋友圈、豆瓣、闲鱼、本地群）存在明显缺口：

- **不知道怎么办**：抓猫、运输、送医、费用、安全风险缺乏结构化指引
- **本地资源难找**：附近哪家宠物医院对流浪猫友好、有无优惠，信息分散
- **故事难传播**：救助过程缺乏「可分享、可接力」的产品载体，难以出圈
- **流程无闭环**：从「发现」到「送医」到「建档/领养/带回家」各环节断裂

### 1.2 产品定位

**捡到猫** 是一款面向「好心捡到猫的路人」的轻量救助引导与社交传播工具。

> **一句话定位**：捡到猫不知道怎么办？发一条动态，附近医院、宠物专车、建档、领养，一条龙。

**核心差异化**：

| 维度 | 竞品常见做法 | 捡到猫 |
|------|-------------|--------|
| 传播 | 纯求助帖 | 梗化叙事 + 进度条 + 分享海报（「要不起」「救了」） |
| 资源 | 全国泛信息 | **本地**宠物医院 + 流浪猫优惠标签 |
| 流程 | 发帖即结束 | 发现 → 送医 → 建档 → 领养/回家 **状态闭环** |
| 出行 | 无 | **一键跳转滴滴宠物专车**，附带目的地与运输提示 |

### 1.3 黑客松目标

在 48 小时内交付 **手机浏览器可访问、可演示、可分享** 的 MVP，验证：

1. 用户愿意发布「捡到猫」动态（带图/定位），在地图中可以显示
2. 「附近医院 + 优惠」信息对决策有实际帮助
3. 选医院后可 **跳转滴滴宠物专车** 完成送医出行闭环
4. 分享链路具备传播潜力（链接 + 梗 + 进度 + 视觉）

**非目标（黑客松不做）**：微信小程序、原生 App、真实支付、滴滴开放平台 API 深度集成、全国医院数据、内容 AI 审核、实名领养平台。

## 2. 目标与用户

### 2.1 产品目标

| 目标 | 说明 | 黑客松验收标准 |
|------|------|----------------|
| 降低决策成本 | 3 分钟内知道「下一步去哪、做什么」 | Demo 主链路 ≤ 5 次点击 |
| 本地资源触达 | 展示最近 3 家友好医院及优惠说明 | 上海种子数据 ≥ 20 家 |
| 出行闭环 | 选医院后一键跳转滴滴宠物专车 | 真机测试跳转成功率 ≥ 1 条可用链路 |
| 传播出圈 | 每条救助可生成可分享内容 | 支持复制链接 + OG 预览图 |
| 安全合规 | 首次使用提示抓猫/运输风险 | 发布前强制阅读安全须知 |

### 2.2 成功指标（Hackathon Demo）

| 指标 | 目标值 |
|------|--------|
| 核心流程完成率 | Demo 账号 100% 跑通 |
| 页面数 | 7 个核心页面可交互 |
| 移动端适配 | iOS Safari / 微信内置浏览器布局正常 |
| 滴滴跳转 | 「叫宠物专车」按钮在真机可唤起滴滴 |
| 分享 | 链接带 Open Graph 预览（标题 + 封面图） |
| 答辩 | 3 分钟完整故事演示 |

### 2.3 用户画像

#### persona A：路人救助者（核心用户）

- **场景**：路边/小区捡到猫，想帮但不确定怎么办、怕花钱、怕被抓伤
- **设备**：手机浏览器（含微信内打开链接）
- **动机**：求助 + 记录 + 希望有人接力
- **痛点**：不知道去哪、不敢抓、担心费用、发了朋友圈没人看、带猫打车被拒

#### persona B：围观接力者

- **场景**：刷到好友/群里的救助链接
- **动机**：点赞、转发、提供医院信息、后续领养
- **行为**：查看进度、评论鼓励、分享扩散

#### persona C：本地资源方（远期）

- 宠物医院、救助站、领养机构
- 黑客松阶段：**静态数据 + 「合作医院」标签**，不做 B 端后台

## 3. 核心用户旅程

### 3.1 主链路（MVP 必做）

```
发现猫 → 发布动态（图/定位/「要不起」）→ 标记「救了」→ 选附近医院
      → 【跳转滴滴宠物专车】→ 更新「已送医」→ 上传检查报告
      → 生成救助档案 → 待领养 / 带回家 → 复制链接分享出圈
```

### 3.2 状态机

救助记录（Rescue）状态流转：

```
discovered   发现/发布      「我捡到猫了，要不起…」
saved        已救助        猫已安全控制（航空箱/暂留）
hospital     送医中        已选医院，前往或已到（含跳转专车记录）
treated      已检查        已上传报告/备注
adoption     待领养        开放领养信息
homeward     已回家        救助者自留或已安置
closed       已结案        归档，仅展示
```

**规则**：

- 状态只能顺序前进或停留在当前态（不可回退，避免 demo 混乱）
- 每次状态变更可选填备注 + 1 张图
- 每次变更自动生成「进度分享文案」
- 点击「叫宠物专车」不强制变更状态，但建议引导用户随后点「已出发送医」

### 3.3 关键场景故事板

**场景 1：要不起，先发声**

> 小王 night 在小区捡到一只橘猫，不敢抓，先拍照发一条：「要不起，求附近姐妹支招」，带模糊定位。帖子出现在同城 feed，有人评论推荐医院。

**场景 2：救了，叫宠物专车去医院**

> 小王把猫放进航空箱，点「救了」→ 系统推荐最近 3 家「流浪猫友好」医院 → 选定医院后点 **「叫滴滴宠物专车」**，自动跳转滴滴 App/小程序的宠物出行页，目的地为医院地址 → 同时可复制地址备用。

**场景 3：建档 + 出圈**

> 医院做完检查，上传报告照片，档案生成。进度 80%，复制链接发到群：「一只橘猫被我捡了，现在等一个家」。

## 4. 功能需求

### 4.1 功能全景

| 模块 | 功能 | 优先级 | 黑客松范围 |
|------|------|--------|------------|
| 账户 | 手机号 / 昵称登录（Web） | P0 | 必做（可 mock 验证码） |
| 动态 | 发布图文 + 定位 | P0 | 必做 |
| 动态 | 同城/附近 feed 流 | P0 | 必做 |
| 救助 | 状态流转（6 态） | P0 | 必做 |
| 地图 | 附近医院列表 + 地图 pin | P0 | 必做（种子数据） |
| 出行 | **跳转滴滴宠物专车** | **P0** | **必做（外链 + 降级）** |
| 档案 | 救助详情 + 报告上传 | P0 | 必做 |
| 分享 | 链接分享 + OG 预览 | P0 | 必做 |
| 安全 | 抓猫/运输风险提示 | P0 | 必做 |
| 评论 | 基础评论 | P1 | 可选 |
| 领养 | 领养意向表单 | P2 | 表单占位 |
| 支付 | 众筹/捐款 | — | 不做 |
| 小程序 | 微信小程序 | — | **本期不做** |

### 4.2 功能详述

#### F1 Web 登录与用户资料

- **登录方式（黑客松）**：手机号 + 短信验证码（可用固定码 `123456` mock）或「昵称快速体验」
- Token 存 `localStorage` / HttpOnly Cookie
- 用户页展示：我发布的救助、我参与的接力（点赞/评论数）

**远期**：微信 OAuth（网页授权）、手机号一键登录。

**验收**：首次进入完成登录/体验入口，再次进入免登录（7 天内）。

---

#### F2 发布救助动态

**入口**：首页 FAB「我捡到猫了」

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 图片 | 图片 1–9 张 | 是（≥1） | `<input type="file" accept="image/*" capture>`，首张为封面 |
| 描述 | 文本 ≤ 500 字 | 是 | 支持快捷模板 |
| 定位 | 经纬度 + 地址 | 是 | 浏览器 Geolocation API；默认模糊到街道/小区级 |
| 标签 | 多选 | 否 | `#要不起` `#已控制` `#受伤` `#幼猫` |
| 安全确认 | 勾选 | 是 | 「我已阅读抓猫安全提示」 |

**快捷模板（一键插入）**：

- 「刚捡到一只猫，要不起，求支招 🐱」
- 「猫已进航空箱，准备送医」
- 「检查完了，等一个家」

**验收**：发布后出现在 feed + 个人页；定位在地图上可见（模糊）。

---

#### F3 Feed 流

- **Tab 1 附近**：按距离排序，默认半径 5km（可调 1/5/10km）
- **Tab 2 最新**：按时间排序
- 卡片展示：封面图、描述摘要、标签、距离、状态徽章、时间
- 下拉刷新 + 上拉加载（黑客松可简化为全量加载）

**验收**：至少 3 条 seed 数据 + 新发布实时可见。

---

#### F4 救助详情与状态流转

**详情页模块**：

1. 头图轮播 + 描述 + 标签
2. **进度条**（可视化 6 阶段）
3. 操作按钮（根据当前状态显示下一步）
4. **出行模块**（选医院后常驻）：「叫滴滴宠物专车」
5. 时间线（每次状态变更的记录）
6. 地图（发现位置 pin）
7. 评论列表（P1）

**状态操作按钮映射**：

| 当前状态 | 主按钮文案 | 点击后 |
|----------|------------|--------|
| discovered | 「猫已安全控制 · 救了」 | → saved |
| saved | 「准备去医院」 | 跳转选医院 |
| hospital | 「已到院 / 已检查」 | → treated，可选上传报告 |
| treated | 「开放领养」/ 「自己带回家」 | → adoption / homeward |
| adoption | 「已找到家 · 结案」 | → closed |
| homeward | 「结案」 | → closed |

**验收**：Demo 账号可从头走到 closed；进度条与 timeline 同步更新。

---

#### F5 附近医院

**入口**：详情页「去医院」/ 首页「附近友好医院」

**列表字段**：

| 字段 | 说明 |
|------|------|
| 名称 | 医院名 |
| 距离 | 基于用户当前位置计算 |
| 地址 | 可复制 |
| 电话 | `tel:` 链接可拨打 |
| 标签 | `流浪猫友好` `TNR` `首诊优惠` `绝育折扣` |
| 优惠说明 | 文本，如「流浪猫绝育 8 折，需说明来源」 |
| 营业时间 | 文本 |

**交互**：

- 列表按距离排序，展示 Top 10
- 地图模式：高德/腾讯 JS SDK pin + 点击出卡片
- 「导航」→ 跳转高德/腾讯/Apple 地图 URI（见附录 C）
- 「叫滴滴宠物专车」→ 见 **F6**

**MVP 数据**：`hospitals.json` 静态文件，≥ 20 条上海样本。

**验收**：选医院后，救助记录写入 `hospital_id`，状态 → hospital；出行模块显示该医院为目的地。

#### F6 滴滴宠物专车跳转（P0 · 必做）

**产品目标**：用户在选定医院后，一键跳转到滴滴的 **宠物出行** 服务（宠物快车 / 宠物专车），减少「带猫被拒载」 friction。

**展示位置**：

- 选医院确认页：主按钮「叫滴滴宠物专车去这里」
- 救助详情页（`hospital` 态）：出行卡片常驻
- 可选：首页底部「带猫出行指南」

**UI 模块**：

```
┌─────────────────────────────────────┐
│ 🚗 带猫去医院                        │
│ 目的地：XX宠物医院（2.3km）          │
│ [ 叫滴滴宠物专车 ]  [ 复制地址 ]     │
│ 提示：请使用航空箱，后排安置          │
└─────────────────────────────────────┘
```

**跳转策略（多级降级，按环境自动选择）**：

| 优先级 | 环境 | 实现方式 |
|--------|------|----------|
| 1 | 微信内置浏览器 | 打开 **URL Link** 跳转滴滴出行小程序宠物出行页（配置项，见附录 D） |
| 2 | 已安装滴滴 App | 尝试 `OneTravel://` / Universal Link 唤起 App（目的地参数尽力携带） |
| 3 | 普通移动浏览器 | 跳转滴滴 App 下载页 / 官方 H5，并 **自动复制目的地地址** 到剪贴板 |
| 4 | 全部失败 | Toast「请手动打开滴滴 App → 首页「宠物出行」→ 粘贴地址」 |

**配置项**（`config/didi.ts` 或环境变量，便于黑客松快速改链）：

```typescript
export const DIDI_CONFIG = {
  // 滴滴出行小程序 AppID（公开信息）
  miniProgramAppId: 'wxaf35009675aa0b2a',
  // 宠物出行页 path（团队需在真机抓包/咨询滴滴确认后填入）
  petTravelPath: '/pages/xxx/index',  // TODO: 开发首日验证
  // 微信 URL Link（在微信内打开，优先使用）
  wechatUrlLink: 'https://wxaurl.cn/xxxxx',  // TODO: 后台生成
  // 备用：小程序 Short Link 文案（用于复制给用户）
  shortLinkFallback: '#小程序://滴滴出行丨打车骑行火车租车代驾/xxxxx',
  // App 唤起（带目的地，不保证直达宠物子页）
  appSchemeTemplate: 'OneTravel://dache/sendorder?dlat={lat}&dlon={lng}&dname={name}',
  // App 未安装时的下载页
  appDownloadUrl: 'https://www.didiglobal.com/download',
};
```

**跳转时携带参数**：

| 参数 | 来源 | 说明 |
|------|------|------|
| `dname` | 医院名称 | 目的地名称 |
| `dlat` / `dlon` | 医院坐标 | 目的地经纬度 |
| `from` | 固定 `jiandaomao` | 来源追踪（query） |

**埋点**（可选 P1）：

- `didi_pet_click`：点击跳转
- `didi_pet_fallback`：走了哪级降级
- `didi_pet_copy_address`：复制地址

**安全与提示文案**（按钮下方固定展示）：

1. 建议使用 **航空箱/猫包**，勿徒手抱猫乘车
2. 猫放 **后排**，勿放副驾驶
3. 首次使用需在滴滴填写 **宠物档案**（类型、体重）
4. 上海等城市可选 **宠物专车**（49 城），全国可用 **宠物快车**
5. 跳转后将离开本平台，行程与费用由滴滴承担

**验收标准**：

- [ ] 选医院后出现「叫滴滴宠物专车」主按钮
- [ ] 微信内打开网页，点击后可跳转至滴滴小程序（或明确降级提示）
- [ ] 系统浏览器点击后可唤起滴滴 App 或下载页
- [ ] 失败时自动复制医院地址，并展示手动操作指引
- [ ] Demo 前在 **iOS + Android 各测 1 次**

**开发首日任务（阻塞项）**：成员 D 负责在真机验证可用跳转链接，填入 `DIDI_CONFIG`；若无法直达宠物子页，降级为「打开滴滴 + 复制地址 + 图文指引进入宠物出行」。

#### F9 安全与合规提示

**首次发布前弹窗**（必须勾选）：

1. 勿徒手抓野猫，建议使用诱捕笼/航空箱
2. 注意抓伤咬伤，及时就医
3. 运输时保持通风，勿长时间密闭
4. 发现明显传染病症状，请联系专业救助机构
5. 本平台仅提供信息参考，不构成医疗建议

**定位隐私**：

- 发布时默认「模糊定位」（偏移 200–500m 或仅显示街道名）
- 提供「仅同城可见精确位置」开关，默认关闭

**第三方跳转声明**：

- 跳转滴滴后适用滴滴用户协议与隐私政策；本平台不介入网约车订单

5. 将 **最终可用链接** 写入 `config/didi.ts`
6. 若均不可直达宠物子页：**接受降级**——打开滴滴首页 + 复制医院地址 + 3 步图文指引

### D.3 前端跳转伪代码

```typescript
async function jumpToDidiPetTrip(hospital: Hospital) {
  const env = detectEnv(); // wechat | ios | android | other
  const payload = { lat: hospital.lat, lng: hospital.lng, name: hospital.name };

  if (env === 'wechat' && DIDI_CONFIG.wechatUrlLink) {
    window.location.href = DIDI_CONFIG.wechatUrlLink;
    return;
  }

  if (DIDI_CONFIG.appSchemeTemplate) {
    const scheme = fillTemplate(DIDI_CONFIG.appSchemeTemplate, payload);
    window.location.href = scheme;
    setTimeout(() => fallbackCopyAndGuide(hospital), 1500);
    return;
  }

  fallbackCopyAndGuide(hospital);
}

function fallbackCopyAndGuide(hospital: Hospital) {
  copyToClipboard(hospital.address);
  showToast('地址已复制，请打开滴滴 App → 宠物出行');
}
```

### D.4 答辩话术（若无法 deeplink 直达）
