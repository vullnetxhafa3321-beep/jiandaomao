## 一、竞品分析

### 1.1 现有流浪宠物救助平台对比

| 平台 | 类型 | 核心功能 | 覆盖范围 | 优势 | 不足 |
|------|------|----------|----------|------|------|
| **宠安家** | 小程序 | 领养发布、申请审核、救助站入驻 | 全国 | 用户量大、领养流程规范 | 偏重"发布-领养"，缺少"捡到后怎么办"的新手引导 |
| **宠物帮领养中心** | 公众号+小程序 | 领养信息汇总、绝育券发放 | 全国 | 合作医院多、医疗资源整合好 | 信息分散、无全流程指引 |
| **易领宠** | 小程序 | 待领养宠物信息展示 | 未明确 | 界面简洁 | 功能单一，仅信息展示 |
| **宠湾** | 小程序 | 救助组织数字化管理 | 未明确 | 面向B端救助站 | 不面向C端普通用户 |
| **萌爪之家(Pawmily)** | 小程序+O2O | 救助地图、领养、公益商城、社区 | 西南地区 | "消费即公益"模式成熟 | 覆盖有限、功能偏重 |
| **深圳市养犬服务** | 政务小程序 | 鼻纹+芯片识别、寻宠、领养匹配 | 深圳 | 政府背书、技术先进 | 仅限深圳、政府主导 |
| **附近有喵** | APP | 爱心商城、救助对接 | 全国 | "以商养善"闭环，获数千万融资 | 需下载APP |
| **老疯杨** | 自媒体+电商 | 内容引流→电商→反哺救助 | 四川 | 年营收6700万、5.2万云认养用户 | 重内容运营、非工具型产品 |

### 1.2 用户对现有方案的吐槽

| 渠道 | 优点 | 痛点 |
|------|------|------|
| **小红书** | 经验分享丰富、真实案例多 | 信息碎片化、无法本地化、广告多、难以验证真伪 |
| **百度** | 搜索方便 | 信息过时、SEO垃圾内容多、无结构化流程 |
| **救助公众号** | 本地化信息 | 更新慢、各号信息不互通、交互差 |
| **宠物医院官网** | 专业 | 大多不标注流浪救助价格、需一个一个打电话问 |

### 1.3 市场空白（你的机会）

> **没有一个产品做「捡到流浪动物 → 第一步怎么办 → 附近去哪治 → 花多少钱 → 自己养还是送救助站」这个全流程 + 本地化的碎片整合。** 现有的竞品要么偏后端（领养发布），要么偏内容（小红书），中间"第一响应人"的角色是真空地带。

## 三、技术栈推荐（非技术人员 · 免费优先）

### 方案A：零代码搭建（最快，推荐黑客松 Demo）

| 工具 | 用途 | 价格 |
|------|------|------|
| **Coze（火山引擎）** | 拖拽工作流 + 大模型知识库 + 直接发布微信小程序 | 按使用量计费，有免费额度 |
| **秒哒（百度）** | 对话式无代码开发，一句话生成小程序 | 免费（每日100秒点） |
| **乔拓云** | 拖拽式小程序搭建，模板丰富 | 199元/年起 |

- **推荐**：用 **Coze** 搭建智能问答 + 信息展示类小程序，配上知识库（救助流程、医院价格、救助站列表），零代码可实现 P0 三个功能。
- **优势**：1小时内可出Demo，Coze 自带大模型能力，知识库可结构化存储医院/救助站数据。
- **局限**：地图定位功能需要确认 Coze 是否支持，可能需配合腾讯地图插件。

### 方案B：AI 辅助编码（更灵活，但需一定学习）

| 工具 | 用途 | 价格 |
|------|------|------|
| **微信开发者工具** | 小程序官方IDE | 免费 |
| **Cursor / Claude Code** | AI 辅助编码 | 各有免费额度 |
| **UniApp + AI** | 跨端小程序框架 | 免费开源 |
| **Supabase / 微信云开发** | 后端数据库 + 云函数 | 免费额度充足 |

- **推荐**：用 **微信云开发** 做后端（免费额度：2GB数据库、5GB存储、云函数10万次/月），前端用 Cursor 或 Claude Code 辅助生成页面代码。
- **优势**：功能不受限、可完全自定义。
- **学习曲线**：需要理解小程序基本结构（约2-3天入门）。

### 方案C：混合方案（推荐！）

| 模块 | 工具 |
|------|------|
| 知识库问答（全流程指南 + FAQ） | **Coze** 知识库 + 大模型 |
| 地图展示（救助站 + 医院） | **微信小程序原生 + 腾讯地图API** |
| AI 品种识别 | **百度AI动物识别API**（500次/天免费） |
| 后端数据管理 | **微信云开发**（免费） |

> **黑客松建议**：方案C 混合——Coze 搞定核心问答体验，小程序原生搞定地图展示，两小时可搭出可演示 Demo。

## 五、核心内容数据（可直接用于产品）

### 5.1 流浪动物救助全流程 SOP

```
第1步：观察判断（3-5分钟）
  → 是流浪还是散养？有无项圈？毛发体态状况？
  → 如果有幼猫且干净安静→不要碰！等4小时看母猫是否回来
  → 紧急伤情（流血/骨折/昏迷）→直接送医

第2步：安全抓捕
  → 厚手套 + 航空箱/纸箱
  → 用食物引诱，不要徒手抓
  → 毛巾从后方包裹，固定四肢

第3步：临时安置
  → 安静避光角落，25-28°C
  → 先不喂食，30分钟后给少量温水
  → 隔离：远离小孩、原住民宠物

第4步：送医检查（必做！）
  → 基础体检：体温、体重、外伤
  → 传染病筛查：猫瘟、猫白血病、猫艾滋
  → 粪便检查（寄生虫）、体内外驱虫
  → 扫描芯片（排查走失宠物）
  → 费用参考：200-500元

第5步：同步寻主（保留1-2周）
  → 拍正脸+全身照
  → 发小区群、宠物店、小红书、微博

第6步：隔离观察（7-14天）
  → 单独房间，每日消毒
  → 不洗澡、不与原住民接触

第7步：决定收养或送救助站
  → 收养：疫苗+绝育+封窗+猫砂盆猫粮
  → 送救助站：联系附近机构、发布领养信息
```

## 九、关键洞察 & 建议

1. **定位差异化明确**：「第一响应人」角色是市场空白——从"捡到"到"解决"，没有一个产品真正整合了这个流程。
2. **内容才是核心竞争力**：SOP 流程 + 本地化医院/救助站数据 + 真实价格参考，是其他竞品做不到的。
3. **全免费技术栈是可行的**：微信云开发 + 百度AI + Coze + 腾讯地图，零成本跑通 MVP。
4. **商业模式有参考**：起步阶段纯公益，后续可通过公益商城、品牌合作、增值服务实现可持续运营。
5. **黑客松Demo建议聚焦北京**：数据先覆盖北京，流程走通后再扩展。
